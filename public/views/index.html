Dodedodo


Dodedodo is a distributed market place that allows you to run applications across a wide range of devices. Suppose, you want to use the camera on your Android phone, connect it to face recognition software on the internet, and have your own door opener based on Arduino to finally open the door for you. This is all possible with off-the-shelf software! But setting it up is really hard.


What is this in general?

Dodedodo uses a standard format to describe software modules. These modules can be connected with each other through a graphical user interface. The modules are found through crawling or when people submit their github repository. The modules run on servers online and on your own devices.

What is this exactly?

Dodedodo will feature as a market place hub for applications that have modules that can run on your Raspberry PI, your Android smartphone, your Arduino do-it-yourself chip, your robots, as well as on servers on the internet. It is extremely powerful at one hand:

  • Automatic generation of glue-code for wide range of middleware: YARP, ROS, ZeroMQ, NodeJS
  • Compiles for many devices: ARM (Android), Blackfin (robots), ARM-HF (Raspberry PI)
  • Allows you to add your own modules
  • Allows deployment on your own server park

It is extremely restricted at the other hand:

  • The modules themselves are in C++
The code of the module itself is currently in C++. This means that this module will run on every platform that has a C++ compiler, which are many. It does not mean that you cannot use other languages. The NodeJS backend is a good example, where glue code is generated that allows you to use the module from Javascript. However, to use that same module also on a robot, we have to enforce the language of the module itself to be C++.


What does this require from the user?

The user will be able to search for cool schemes that exist out of multiple modules. Those modules have to be dragged to devices the user owns. So, this is slightly more involved than installation of a normal app on a smartphone. On all the devices the user registers there will be a small application that allows modules to be launched, stopped, and connected with other modules. This application is called an AIM agent. The user does not have to worry about firewalls, or anything else, just installing the AIM agent on a Raspberry PI, or an Android smartphone is all.
